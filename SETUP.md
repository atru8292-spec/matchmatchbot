# Подготовка к работе с Claude Code — MatchMatch bot

## 1. Установить Claude Code (один раз на своём компе)

```bash
# нужен Node.js 18+
node --version

# установка
npm install -g @anthropic-ai/claude-code

# проверка
claude --version

# первый запуск — залогинишься через браузер (Anthropic аккаунт)
claude
```

## 2. Два варианта где вести проект

### Вариант A — код лежит на СЕРВЕРЕ, правишь по SSH (рекомендую для бота)

Бот всё равно должен крутиться на серваке 24/7, так что код держим там.

```bash
# заходишь на сервер
ssh root@ТВОЙ_IP_СЕРВЕРА

# создаёшь папку проекта
mkdir -p /opt/matchmatch-bot
cd /opt/matchmatch-bot

# инициализируешь git (для версий и откатов)
git init

# ставишь Claude Code прямо на сервер (если Node есть)
npm install -g @anthropic-ai/claude-code

# запускаешь Claude Code внутри папки проекта
claude
```

### Вариант B — код на своём компе, деплой на сервер потом

```bash
# на своём компе
mkdir matchmatch-bot
cd matchmatch-bot
git init
claude
```

Правишь локально, тестишь, потом `git push` → на сервере `git pull` → рестарт.
Проще дебажить, но нужен деплой-шаг. Для начала можно так, позже перенести.

## 3. Первым делом в Claude Code — дать контекст

Когда запустишь `claude` в папке, скажи ему:

```
Прочитай CLAUDE.md и все файлы в .claude/agents/.
Затем составь план миграции n8n workflow на Python.
Не пиши код пока я не одобрю план.
```

## 4. Золотые правила работы (чтобы не было багов вслепую)

1. **Всегда сначала ПЛАН, потом код.** Claude Code имеет plan mode — жми Shift+Tab дважды, включается планирование. Он покажет что собирается делать, ты одобряешь.

2. **"Никаких «готово» без доказательств".** Это правило зашито в CLAUDE.md — Claude обязан ЗАПУСТИТЬ тест и показать результат, а не сказать "работает".

3. **Используй субагентов для проверки.** После каждой фичи говори: "запусти code-reviewer агента на这и изменения". Он проверит в отдельном контексте и отчитается.

4. **Ревьюй spec до кода.** Поменять план — бесплатно. Переписать код — часы. Читай что Claude предлагает ПЕРЕД тем как он пишет.

5. **Модели:** Opus для планирования сложного, Sonnet для написания кода. Переключение: `/model opus` или `/model sonnet`.

## 5. Полезные команды внутри Claude Code

```
/model opus       # переключить на Opus (думать)
/model sonnet     # переключить на Sonnet (кодить)
/agents           # управление субагентами
/clear            # очистить контекст (между задачами)
Shift+Tab дважды  # включить plan mode
```
