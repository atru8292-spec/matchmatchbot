# MatchMatch WhatsApp Bot — проект

## Что это
AI-бот "Anna" отвечает на входящие WhatsApp-лиды агентства знакомств MatchMatch (CDMX).
Пишет от имени Anna (основательница), лид думает что общается с ней.
Мигрируем с n8n на Python-бэкенд.

## Стек
- Python 3.11+, FastAPI (вебхук-сервер)
- Supabase (Postgres) — база лидов, сообщений, сценариев
- OpenAI — генерация ответов (GPT-4.1) + транскрибация голосовых (Whisper)
- Wazzup24 — шлюз WhatsApp (приём/отправка)
- Telegram Bot — алерты об ошибках и эскалациях

## Архитектура (модули)
- `main.py` — FastAPI, эндпоинты вебхуков
- `normalize.py` — разбор входящего payload (текст/фото/голос/видео)
- `db.py` — все запросы к Supabase
- `ai.py` — вызов OpenAI + RAG по сценариям
- `sender.py` — разбивка ответа на сообщения, задержки, typing, отправка
- `escalation.py` — алерты в Telegram
- `debounce.py` — склейка серии быстрых сообщений от лида

## Жёсткие правила (соблюдать всегда)

### Верификация — САМОЕ ВАЖНОЕ
НИКОГДА не говори "готово / работает / тесты прошли" без свежего запуска.
Перед любым заявлением о статусе:
1. ЗАПУСТИ команду (pytest / python -m ...) прямо сейчас, не кэш
2. ПОКАЖИ реальный вывод
3. Только потом делай вывод
Если не можешь проверить — не заявляй что работает.

### Код
- Никаких секретов в коде. Все ключи в .env (Wazzup, Supabase, OpenAI, Telegram)
- Каждая функция с обработкой ошибок — если падает, лог + алерт в Telegram
- Комментарии на русском, код на английском
- Простые решения предпочтительнее умных. Читаемость важнее краткости

### Бизнес-логика бота (не нарушать)
- Бот НЕ отвечает клиентам агентства (текущим и новым) — они идут к Anna лично.
  Есть whitelist номеров, для них бот молчит
- Бот НЕ даёт скидок и НЕ даёт телефоны девушек
- Бот в чате называет только Starter $1,400/мес. Standard/VIP — только на видеозвонке у Anna
- Фильтры: возраст 28-65, только soltero, престижная профессия (AI решает по контексту), нужно фото
- Блокирует и молчит: возраст вне 28-65, не холост, casual/escort, агрессия, неприемлемое фото

### Антибан WhatsApp
- Задержки между сообщениями (2-8 сек по длине + рандом)
- Индикатор "печатает" перед каждым сообщением
- Рассылки по старой базе — порциями 20-30/день, не всё разом
