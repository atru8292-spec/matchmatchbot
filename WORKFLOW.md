# Как вести работу с Claude Code — рабочий цикл

Это твой чеклист чтобы Claude Code делал всё продуманно, тестил себя и отчитывался.

## Полный цикл одной задачи (по шагам)

### Шаг 1. План (Opus)
```
/model opus
```
Скажи что нужно. Claude составит план. НЕ давай писать код сразу.
Читай план. Спрашивай "а что если X?". Менять план дёшево, менять код дорого.

### Шаг 2. Одобрение
Когда план норм — "ок, приступай". Переключись на Sonnet для кода:
```
/model sonnet
```

### Шаг 3. Реализация
Claude пишет код модуль за модулем. После каждого модуля — не дальше, а проверка.

### Шаг 4. Тесты (субагент)
```
Запусти tester агента на этот модуль
```
Он напишет тесты, ЗАПУСТИТ их, покажет что упало. Не примет "работает" на веру.

### Шаг 5. Ревью (субагент)
```
Запусти code-reviewer на изменения
```
Проверит баги, обработку ошибок, бизнес-логику в отдельном контексте.

### Шаг 6. Фикс
Если ревьюер/тестер нашли проблемы — Claude чинит, потом ПЕРЕзапускает тесты.

### Шаг 7. Коммит
```
Закоммить с понятным сообщением
```
Git = откат если что-то сломается.

### Шаг 8. Раз в этап — улучшения
```
Запусти improver агента, посмотри что можно улучшить
```

## Фразы которые заставляют работать правильно

| Хочешь | Говоришь |
|---|---|
| Не лепить код сразу | "Сначала план, код не пиши пока не одобрю" |
| Реальную проверку | "Запусти тесты и покажи вывод, не говори работает без запуска" |
| Чистый контекст | "Используй субагента чтобы изучить X" |
| Ревью | "Запусти code-reviewer на эти изменения" |
| Найти баги | "Запусти tester, покажи что падает" |
| Улучшения | "Запусти improver, дай приоритизированный список" |
| Между задачами | "/clear" (очистить контекст) |

## Красные флаги — когда Claude халтурит

- Говорит "тесты прошли" но ты не видела запуска → "покажи реальный вывод pytest"
- Говорит "готово" без доказательств → "чем докажешь? запусти и покажи"
- Начал читать 100 файлов, контекст пухнет → "используй субагента для этого"
- CLAUDE.md разросся, Claude игнорит правила → подрежь его, оставь только важное

## Порядок миграции бота (крупными блоками)

1. Скелет: FastAPI + вебхук который принимает от Wazzup и логирует
2. БД-слой: подключение к Supabase, чтение/запись лидов и сообщений
3. Нормализация payload (текст/фото/голос)
4. Debounce (склейка быстрых сообщений)
5. AI-ядро: OpenAI + RAG по сценариям + промпт Anna
6. Отправка: разбивка + typing + задержки
7. Фото-ветка: скачать → Supabase Storage → Vision-проверка
8. Голосовые: транскрибация Whisper + алерт Anna
9. Эскалация + все ошибки → Telegram
10. Whitelist клиентов (бот молчит для них)
11. Рассылка по старой базе (порциями)
12. Деплой: systemd, автоперезапуск, логи

Каждый блок = отдельная задача по циклу выше (план → код → тест → ревью → коммит).
Не делай всё разом. Один блок, проверила, работает — следующий.
